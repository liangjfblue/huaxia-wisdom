## 庖丁解牛 — 纹理拆解路线图

### 核心理念

庖丁解牛不是快，是"顺着纹理走"。牛有筋骨、肯綮（关节）、纹理——硬砍会伤刀、费力、还慢。任务也一样：有硬骨头、有关键节点、有自然规律。

关键不是"更努力"，而是找到任务的纹理，顺着走。

### 输出结构

```
🐂 牛的结构分析

骨头（硬难点）：
[必须啃的硬骨头——无法绕过，但可以最小化接触面]

肯綮（关键节点）：
[搞定这一个，其他都顺了的关节点]

纹理（可顺势处）：
[本来就有的缝隙——现成工具、已有流程、可复用的经验]

🔪 解牛路线图

Step 1: 从______下刀
[最省力的切入点——找一个已有的、可复用的起点]

Step 2: 避开______
[绕行方案——可以不碰的硬骨头，为什么可以绕]

Step 3: 直取______
[关键突破点——搞定肯綮]

Step 4: 余势推及______
[顺势完成的后续——前面搞定后，这些自然就完成了]

📜 庄子原话
"依乎天理，批大郤，导大窾，因其固然。技经肯綮之未尝，而况大軱乎！"
[附一句现代解读]

💡 你的任务不是______，而是______。
[把用户的任务重新表述为一个"顺着纹理走"的版本]
```

### 原话储备

- "依乎天理，批大郤，导大窾，因其固然。技经肯綮之未尝，而况大軱乎！" — 《庄子·养生主》
- "臣之所好者道也，进乎技矣。" — 《庄子·养生主》
- "方今之时，臣以神遇而不以目视，官知止而神欲行。" — 《庄子·养生主》

### 示例

用户：庖丁解牛分析一下，我要写一个50页的项目方案

🐂 **牛的结构分析**

**骨头**：数据收集和整理（最耗时，但不是最关键的）
**肯綮**：核心论点确定（定了这个，其他都好写）
**纹理**：公司已有季度报告模板、去年的类似方案、竞品公开资料

🔪 **解牛路线图**

**Step 1**：从"去年方案"下刀（已有结构可复用80%，这是最大的纹理）
**Step 2**：避开"从零收集数据"（让数据分析同事先跑一轮，用现有报告，不自己挖）
**Step 3**：直取"核心论点"（先花2小时定大纲，确定3个核心观点，不动细节）
**Step 4**：余势推及"填充案例"（论点定了，找案例就有方向，一边写一边补数据）

📜 **庄子原话**
"依乎天理，批大郤，导大窾，因其固然。技经肯綮之未尝，而况大軱乎！"
—— 顺着事物的自然结构，避开硬骨头，从缝隙走。连关节都没碰到，何况骨头？

💡 你的任务不是"写50页"，而是"找到去年的方案，换掉核心数据和案例"。

## 无为而治 — 干预审计清单

### 核心理念

"无为"不是什么都不做，而是**不做无效的事**。很多时候你忙得要死但没成果，不是因为你不够努力，而是因为你做了太多"看起来在做事但其实没用"的事。

这个框架帮你做一次干预审计：哪些事做了等于没做，甚至更糟？

### 输出结构

```
📋 干预审计

你正在做的事：
[根据用户描述，列出他正在干预/花精力的具体事项]

无效干预（做了反而更差）：
[识别出来 + 为什么无效 + 建议放手]
- [具体事项] → 无效原因：______ → 建议：______

低效干预（做了但性价比低）：
[识别出来 + 为什么低效 + 替代方案]
- [具体事项] → 低效原因：______ → 替代：______

该做的干预（真正值得花力气的）：
[1-2件真正重要的事——这些才值得你全力以赴]

放手实验：
[建议用户尝试一周不碰某件事，观察会发生什么。
用具体时间：从X到X，不碰Y，看结果]

📜 老子原话
[选一句]

💡 口诀
[四句押韵]
```

### 原话储备

- "太上，不知有之。" — 《老子》
- "为无为，则无不治。" — 《老子》
- "治大国，若烹小鲜。" — 《老子》
- "夫唯不争，故天下莫能与之争。" — 《老子》

## 循名责实 — 名实对照表

### 核心理念

韩非子说"循名责实"——跟着名字去追究实际。意思是：不要看包装，看本质。

这个框架特别适合评估人（招聘、合作）、评估项目（进展汇报）、评估自己（简历 vs 能力）。

### 输出结构

```
🔍 名实对照

名（表面/声称）    →  实（真相/结果）
[头衔/项目名/自述] → [实际产出/实际状态]

具体对照：
[对方说了什么]     → [对方做了什么]
[看起来像什么]     → [实际上是什么]
[承诺了什么]       → [交付了什么]

需要追问的问题：
[1-2个验证"实"的具体问题——直接问就能试出真假]

决策建议：
[基于"实"而非"名"的判断]

📜 韩非子原话
[选一句]

💡 口诀
[四句押韵]
```

### 原话储备

- "循名实而定是非，因参验而审言辞。" — 《韩非子》
- "人主将欲禁奸，则审合刑名。" — 《韩非子》
- "听其言必责其用，观其行必求其功。" — 《韩非子》

## 破名辨实 — 概念拆解器

### 核心理念

名家公孙龙说"白马非马"——白马和马不是同一个概念。这听起来像诡辩，但内核是**打破概念的固化**：我们常常被一个"名字"困住，以为名字就是事物本身。

"我是内向的人""这个行业没前途""我是技术出身做不了管理"——这些都是"名"，不是"实"。你被标签绑架了，所以看不见真实情况。

这个框架帮你**拆掉标签，还原事实**。

### 输出结构

```
🏷️ 概念拆解

你给自己的标签（名）：
[用户给自己/他人/事物贴的标签——
"我是XX的人""这个事是XX""他是XX类型的"]

标签背后的事实（实）：
[去掉标签后，事实是什么——
"内向"可能只是"不喜欢无效社交"
"行业没前途"可能只是"你所在的公司没前途"]

名和实的差距：
[标签让你忽略了什么——
因为"我是技术人员"所以不学管理？
因为"这个赛道成熟了"所以不做了？]

重新定义：
[去掉标签后的真实描述——
不是"我是XX的人"，而是"我在XX方面需要提升"
不是"这个事不可能"，而是"这个事我没找到方法"]

行动解放：
[去掉标签后，你可以做什么——
之前因为标签而不敢做的事，现在可以怎么开始]

📜 名家原话
[选一句]

💡 口诀
[四句押韵]
```

### 原话储备

- "白马非马。" — 公孙龙
- "离坚白。" — 公孙龙（石头的"白"和"坚"是两个不同的属性）
- "指物论。" — 公孙龙（"指"不是"物"，概念不是事物本身）
- "名者，实之宾也。" — 《庄子》（名字是事实的附庸）

## 集大成者 — 跨界整合图

### 核心理念

杂家吕不韦编《吕氏春秋》，核心思想是**博采众长、融会贯通**——不执着于某一家之说，而是把各家的优势整合在一起。

现代人容易陷入"专业壁垒"：我是做产品的、他是做技术的、那是做运营的——井水不犯河水。但真正的突破往往在跨界处：把A领域的方法论用到B领域，把C行业的经验搬到D行业。

这个框架帮你找到**跨领域借鉴的突破口**。

### 输出结构

```
🌐 跨界整合

你当前的问题/领域：
[用户面临的具体问题——先说清楚]

你已有的知识和经验：
[用户当前领域/专业的方法论和经验]

可借鉴的"隔壁领域"：
├─ 领域A：[什么领域 + 为什么相关]
│  └─ 可借鉴的方法：[这个领域解决类似问题的做法]
├─ 领域B：[什么领域 + 为什么相关]
│  └─ 可借鉴的方法：[这个领域解决类似问题的做法]
└─ 领域C：[如果有的话]

跨界迁移方案：
[把"隔壁"的方法论，翻译成"你这里"可执行的方案——
关键是找到两个领域的结构相似性]

整合检查：
[跨界借鉴时容易犯的错——
比如：生搬硬套（忽略领域差异）、水土不服（方法不匹配当前环境）]

第一步：
[今天就可以做的一个小实验——验证跨界借鉴是否可行]

📜 杂家原话
[选一句]

💡 口诀
[四句押韵]
```

### 原话储备

- "兼听则明，偏信则暗。" — 《资治通鉴》（虽然不是吕不韦说的，但精神一致）
- "天下无粹白之狐，而有粹白之裘，取之众白也。" — 《吕氏春秋》（没有纯白的狐狸，但有纯白的裘皮——集合众人的优点）
- "物固莫不有长，莫不有短。人亦然。故善学者，假人之长以补其短。" — 《吕氏春秋》
- "以众智为智，以众能为能。" — 《吕氏春秋》
